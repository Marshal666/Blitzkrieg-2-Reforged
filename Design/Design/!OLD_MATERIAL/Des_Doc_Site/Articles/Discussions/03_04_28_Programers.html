By Yuri V. Blazhevich
Monday, April 28, 2003

Undo делаем не через манипулятор. Манипулятор про Undo ничего не знает.
Для этого необходимо внести изменения в манипулятор. Изменения
следующие: при добавлении пары (строка - значение) в манипулятор,
возвращается iterator, где перечисляются новые (привнесенные) пары
(строка - значение).

При работе с определенным манипулятором мы записываем последовательность
классов-команд в буфер Undo. Каждый такой класс содержит код и данный по
прямой команде для манипулятора и, дополнительно, обратной команде
манипулятора. Пример: прямая команда - добавить пару ("objects.[1]",
указатель на новый обьект), обратная команда - удалить пару
("objects.[1]", указатель). Дополнительно необходимо записывать
дополнительную информацию по визуализации.

При работе с базой есть два режима работы:
  1. Только для чтения.
  2. Для чтения и изменения.
По умолчанию выбирается первый режим, при необходимости внесения
изменения данные перечитываются и блокируются на изменения другим
пользователем.

Есть мысль дополнительно в базе данных для каждого типа записи хранить
права по умолчанию на первый просмотр такого типа данных, всех
пользователей различать правам по умолчанию (дизайнеры, программисты,
художники и .т.д.).
